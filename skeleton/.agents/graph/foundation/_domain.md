# Layer 0 · 基础设施层（Foundation）

> 框架运行的底层基石 — 所有上层能力的物理支撑。

## 层级定位

基础设施层提供框架运行所需的**底层传输和存储机制**。这些组件不直接暴露业务语义，但缺少任何一个都会导致上层功能无法工作。

## 包含节点

| 节点               | 描述              | 关键实现                  |
| ------------------ | ----------------- | ------------------------- |
| mcp-transport      | MCP Server 通信层 | stdio 传输、工具/资源注册 |
| config-system      | 配置加载与降级    | config-loader、默认值回退 |
| file-persistence   | 文件系统持久化    | .trellis/ 目录约定        |
| git-integration    | Git 版本控制集成  | 状态查询、Worktree        |
| manifest-mechanism | Manifest 驱动升级 | SHA-256 哈希、智能合并    |

## 层间关联

- **向上服务** → Layer 1 数据层的所有存储操作依赖文件系统和配置加载
- **向上服务** → Layer 2 能力层的任务管理、质量控制依赖 MCP 传输和 Git 集成
- **横向协作** → 本层节点之间低耦合，各自独立运作
