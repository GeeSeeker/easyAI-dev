# Worker 轻量修复 Workflow (quick-fix)

## 适用场景

当任务的 `assign_strategy` 为 `"quick-fix"`，或任务描述中包含"轻量修复"时，Worker 将自动进入此模式。
该模式适用于：修改范围小、影响面可控的 Bug 修复、简单文案修改、配置微调等。

## 流程步骤（最大 6 步）

1. **约束集读取**
   - 快速提炼修复目标和文件范围（不强制写长篇执行计划）。

// turbo 2. **定位与修改**

- 直奔目标文件，进行原子化代码修改。

// turbo 3. **编译与局部验证**

- 运行相关的局部按需测试或构建命令，确保功能恢复且无明显回归。

// turbo 4. **简化自检与输出**

- 运行复用 `worker-check` 简化版流程。
- 不强制要求全量三标记，允许相关类别标记为 `_NA`。
- 生成简版的 `dev/verification.md`（仅记录实际执行的测试和手动验证说明）。

// turbo 5. **代码提交**

- 自动包含修改文件和自检报告，执行 `git commit`。

// turbo 6. **提交验收**

- 调用 `task_transition`，将任务状态转移给 PM。
