# 文档简化与渐进展开

> 将复杂文档提炼为人类友好版本，用户按需指定章节深入展开。

## 触发条件

1. **自动触发**：AI 产出或引用长度超过 100 行的技术文档时，主动建议简化
2. **用户触发**：用户主动请求「简化这个文档」或「给我一个概要版本」
3. **仅限项目文件**：默认只简化项目内文件；用户可主动请求简化外部内容

## 执行路径

本规范通过以下工作流 hook 被激活：

| 触发点      | 位置                                  | 行为                                                         |
| ----------- | ------------------------------------- | ------------------------------------------------------------ |
| 会话收工    | `common-session-close` Step 4 第 7 条 | 检查本会话产出的长文档，按简化规则创建简化版并维护双向链接   |
| Worker 产出 | `worker-implement` 完成条件           | Worker 产出超过 100 行的文档时，在 `worker-check` 前建议简化 |
| 用户请求    | 随时                                  | 用户说「简化这个文档」时，AI 按本规范执行                    |

## 简化规则

### 简化版结构

```markdown
<!-- source: relative/path/to/original.md -->

# {文档标题}（简化版）

> 源文档：[original.md](relative/path/to/original.md) | 简化日期：YYYY-MM-DD

## 核心要点

- 要点 1（一句话概括）
- 要点 2
- ...

## 章节索引

| #   | 章节     | 概要         | 展开指令        |
| --- | -------- | ------------ | --------------- |
| 1   | {章节名} | {一句话概要} | 「展开第 1 章」 |
| 2   | {章节名} | {一句话概要} | 「展开第 2 章」 |
```

### 源文档标注

在源文档顶部（frontmatter 之后、正文之前）添加：

```markdown
<!-- simplified: relative/path/to/简化版.md -->
```

## 渐进展开

1. 用户指定章节后（如「展开第 2 章」），AI **原地替换**该行的概要为完整内容
2. 展开后的内容保留在简化版中，不再缩回
3. 每次展开更新简化版文件

## 输出位置

- 简化版默认输出到 `.docs/guides/`
- 文件名格式：`{原文件名}-简化版.md`

## AI 行为规范

1. 简化不等于删减 — 确保核心决策和约束在简化版中可见
2. 技术术语保留原文，但附带中文释义
3. 代码示例只保留最关键的一个，其余归入展开章节
4. 双向链接必须在两个文件中同时维护
