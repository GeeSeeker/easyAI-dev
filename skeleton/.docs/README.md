# .docs/ — 意图与蓝图空间

> 这里是 easyAI 框架的**面向人与大局的空间**。记录业务目标、头脑风暴、宏观架构蓝图。

## 目录结构

```text
.docs/
├── notes/             # 纯草稿场区（AI 与用户的白板）
├── refs/              # 外部参考资料库（各种喂料、对接文档）
├── design/            # 正式架构与规划中心
│   ├── planning/      # 具体且固定的长线项目规划、Roadmap、里程碑总结
│   └── features/      # 具体特性的技术架构设计与蓝图
└── archive/           # 归档（失效草稿与过期设计图纸）
```

## 子文件夹详细说明

| 子文件夹           | 用途                                                     | 生命周期与归档规则                                                                    |
| ------------------ | -------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| `notes/`           | 纯粹的发散思考、备忘录。文件须带 `user-` 或 `pm-` 前缀。 | 一旦得出肯定结论，原稿必须移入对应 `archive/` 或删除，禁止长期堆积。                  |
| `refs/`            | 外部知识库、原始对线文档、竞品分析。                     | 仅作设计输入源。失去时效后，由“进度归档”清理至 `archive/`。                           |
| `design/planning/` | 全局路线图、大版本迭代总表、阶段里程碑总结大纲。         | 长期保留。                                                                            |
| `design/features/` | 具体功能的 API 设计图、数据流转蓝图。                    | 完工后仅在文件名前打 `[DONE]` 标记；废弃方案打 `[OBSOLETE]` 标记并移动至 `archive/`。 |
| `archive/`         | 各目录专属的“就地养老”归档区。                           | 存放作废思路、过期图纸、淘汰参考资料。                                                |

## 三大空间核心断言（沉淀隔离原则）

| 空间             | 面向       | 核心特征                                                                                          |
| ---------------- | ---------- | ------------------------------------------------------------------------------------------------- |
| `.docs/`         | 用户与大局 | **意图空间**：这里全是一次性的建筑蓝图与头脑风暴。                                                |
| `.trellis/`      | AI Worker  | **契约空间**：存放绝对的跨功能复用规约、约束集。凡是普适技术决策，**绝不允许**长期留在 `.docs/`。 |
| `.tmp/ / .test/` | 环境       | **业务与瞬态空间**：`.tmp` 禁止放文档，为绝对丢弃区；`.test` 为业务自有测试报告区。               |
